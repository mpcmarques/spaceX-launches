# claude.md

## Project

This project represents a react website that fetches launches from SpaceX.

## Code Style
- The code formatter that should be used is Prettier (configured in .prettierrc.js).
- For React components, always create one for file. All React components files names should have capital case and have default exports.

## Key files
- `src/index.tsx` - React project entry file.
- `src/types/index.tsx` - Types definitions.

## Commands
- `yarn install` to install project dependencies.
- `yarn build` to build the library.
- `yarn start` to run the project locally in development mode.
- `tarn test` to run unit tests.

## Conventions
- You should always use `yarn` as the project package manager.
- Any code added to the project should be formatted using the project code formatting style.
- All new features should have test cases created to cover them.
- Before pushing any code, should first run tests to assert that it didnt break any existing test case.
